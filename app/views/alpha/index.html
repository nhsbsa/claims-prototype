The purpose of this work is to explore how individual credit notes can be processed using the same interaction patterns and design principles established for initial invoice processing. Reply to contestation - Non-RINA - Submission shell iterations

During user research, participants highlighted that while the system prompts users to enter the total amount withdrawn, it does not request the total amount contested. The absence of a visible overall contested amount was identified by several users as a barrier to accurately reviewing and confirming their entries.

In response to this feedback, we introduced an additional field on the details capture screen for Non-RINA claims, allowing users to input the total amount still requested. This value will be displayed later in the journey to support more accurate review and assurance.

For RINA claims, the system can calculate the remaining contested amount automatically, so no changes were required in that workflow.
